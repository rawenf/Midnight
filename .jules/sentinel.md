## 2024-05-18 - Stored XSS via Firestore User Document Fields
**Vulnerability:** A `source` field from a user-created `pin` document in Firestore was being rendered directly into the `href` attribute of an `<a>` tag without sanitization. This allowed an attacker to inject `javascript:` URIs.
**Learning:** Because the Firestore security rules allow authenticated users to create and update their own `pin` documents with arbitrary data (e.g. `{"source": "javascript:alert(1)"}`), any document fields rendered into dangerous contexts (like `href`, `src`, or `dangerouslySetInnerHTML`) represent a critical Stored XSS vector in client-side applications using Firebase.
**Prevention:** Always sanitize URLs fetched from user-controllable databases before placing them in `href` or `src` attributes. Use protocol-relative checks (`startsWith('http://') || startsWith('https://')`) or proper URL parsing libraries to enforce safe protocols. Also ensure that `target="_blank"` links include `rel="noopener noreferrer"`.
